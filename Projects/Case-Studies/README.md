# Case Studies

## Descripción

La carpeta **Case Studies** reúne investigaciones técnicas desarrolladas sobre escenarios reales o simulados dentro del laboratorio de ciberseguridad.

Cada caso documenta el proceso completo de análisis realizado para comprender un evento, validar una alerta o investigar una actividad potencialmente sospechosa utilizando la telemetría recopilada por **Wazuh SIEM**.

El objetivo es demostrar la capacidad de análisis, razonamiento técnico y documentación de incidentes mediante una metodología estructurada.

---

# Objetivos

Los **Case Studies** tienen como finalidad:

- Analizar eventos de seguridad.
- Documentar investigaciones técnicas.
- Comprender el comportamiento de procesos y servicios.
- Validar alertas generadas por el SIEM.
- Mejorar la capacidad de análisis.
- Construir experiencia práctica basada en evidencia.

---

# Alcance

Cada investigación puede estar relacionada con:

- Procesos de Windows.
- Procesos de Linux.
- PowerShell.
- LOLBins.
- Sysmon.
- Windows Event Logs.
- auditd.
- Alertas de Wazuh.
- MITRE ATT&CK.
- Casos de Incident Response.

# Metodología

Todos los casos documentados seguirán una estructura uniforme.

## Estructura del Caso

- Resumen Ejecutivo.
- Objetivo.
- Escenario.
- Fuente de Información.
- Análisis.
- Hallazgos.
- Evaluación.
- Conclusión.
- Mapeo MITRE ATT&CK.
- Competencias Desarrolladas.
- Lecciones Aprendidas.

Esta metodología permite mantener consistencia entre todas las investigaciones realizadas dentro del laboratorio.

# Tipos de Investigaciones

Algunos ejemplos de investigaciones que formarán parte de esta sección son:

- Ejecución de **cmd.exe** desde **PowerShell**.
- Uso de **LOLBins**.
- Creación de procesos sospechosos.
- Persistencia mediante tareas programadas.
- Ejecución de **rundll32.exe**.
- Ejecución de **regsvr32.exe**.
- Uso de **certutil.exe**.
- Actividad relacionada con **PsExec**.
- Cambios en servicios del sistema.
- Modificaciones del Registro de Windows.

Cada caso será documentado de forma independiente.

---

# Objetivo del Portafolio

Los **Case Studies** representan la aplicación práctica de los conocimientos adquiridos durante el desarrollo del laboratorio.

Cada nueva investigación permitirá ampliar el portafolio, documentar nuevas técnicas y fortalecer las capacidades de análisis e investigación dentro del entorno de **Blue Team**.
