# Linux Investigations

## Descripción

La carpeta **Linux Investigations** reúne las investigaciones técnicas realizadas sobre eventos de seguridad generados en sistemas **Linux**, utilizando la telemetría recopilada por **Wazuh SIEM**, **auditd**, **Syslog** y **Journalctl**.

Cada investigación documenta el proceso de análisis aplicado para comprender el comportamiento observado, validar alertas y determinar si una actividad corresponde a una operación legítima o potencialmente maliciosa.

---

# Objetivos

Esta sección tiene como finalidad:

- Analizar eventos de seguridad en sistemas Linux.
- Documentar investigaciones técnicas.
- Comprender el comportamiento de procesos y servicios.
- Validar alertas generadas por el SIEM.
- Correlacionar eventos provenientes de múltiples fuentes.
- Fortalecer las capacidades de análisis e investigación.

---

# Alcance

Las investigaciones podrán incluir escenarios relacionados con:

- SSH.

- Syslog.
- Journalctl.
- Bash.
- Cron.
- systemd.
- Gestión de usuarios.
- Procesos.

Cada caso será documentado de forma independiente.

# Tipos de Investigaciones

Algunos ejemplos de investigaciones que formarán parte de esta sección son:

- Inicio de sesión mediante SSH.
- Uso de sudo.
- Creación de nuevos usuarios.
- Modificación de permisos.
- Persistencia mediante cron.
- Ejecución de comandos administrativos.
- Cambios en servicios del sistema.
- Actividad inusual de procesos.
- Escalamiento de privilegios.
- Accesos remotos.

Cada investigación será documentada de forma independiente y basada en eventos generados dentro del laboratorio.

# Objetivo del Portafolio

Las investigaciones documentadas en esta sección representan la aplicación práctica de los conocimientos adquiridos durante el desarrollo del laboratorio, permitiendo demostrar experiencia en el análisis de eventos, validación de alertas e investigación de incidentes en sistemas Linux.
