# TH-001 - Threat Hunting: PowerShell EncodedCommand

## Objective

Validate whether the execution of PowerShell using the `-EncodedCommand` parameter corresponds to an isolated event or represents a broader malicious activity within the monitored environment.

---

# Hunting Hypothesis

PowerShell executed with the `-EncodedCommand` parameter is frequently associated with defense evasion, payload obfuscation and malicious script execution.

Hypothesis:

> If an attacker executed an encoded PowerShell command, additional executions or related indicators should also exist within the environment.

---

# Environment

Platform

- Windows 11

# Findings

The investigation determined:

- Only three PowerShell executions used the EncodedCommand parameter.
- Only one execution generated the custom detection rule.
- No additional hosts were affected.
- No evidence of lateral movement was identified.
- No persistence mechanisms were observed.
- The activity remained isolated within the laboratory endpoint.

# Threat Hunting Assessment

The investigation did not identify evidence suggesting an active compromise.

The encoded PowerShell execution corresponded to a controlled laboratory simulation used to validate detection capabilities.

No additional suspicious PowerShell activity was identified during the review.

---

# Lessons Learned

PowerShell is commonly executed by legitimate administrative tasks.

However, the use of the `-EncodedCommand` parameter should always receive additional investigation because it is widely used by attackers to hide malicious commands.

Threat hunting allows analysts to determine whether a single alert is an isolated event or part of a larger intrusion.

---

# Recommendations

- Continue monitoring EncodedCommand executions.
- Correlate PowerShell activity with network connections.
- Validate parent-child process relationships.
- Review Script Block Logging events when available.
- Maintain detection rules for PowerShell abuse.

---

# Conclusion

The hunting activity confirmed that the detected execution was limited to the laboratory simulation.

No additional indicators of compromise were identified, allowing the event to be classified as an isolated and expected activity generated during detection validation.
